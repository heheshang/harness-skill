# 需求规格说明书

## 背景

当前价格系统只支持单一价格规则，无法满足不同渠道、不同用户群体的差异化定价需求。需要新增价格过滤规则功能，支持按渠道、用户等级、地域等维度配置差异化价格策略。

## 需求描述

支持按多维度配置价格过滤规则，实现差异化定价：
- 支持按渠道维度过滤（线上/线下/特定渠道）
- 支持按用户等级过滤（普通用户/VIP/ SVIP）
- 支持按地域维度过滤（华东/华南/全国）
- 支持规则优先级设置（数字越小优先级越高）
- 支持规则生效时间段设置

## 变更范围

### 涉及的模块
- `core/service`：新增 PriceRuleFilterService 价格过滤服务
- `core/flow`：新增价格过滤流程组件
- `dal/mapper`：新增 PriceRuleMapper 查询方法
- `common/constant`：新增价格规则相关常量

## 影响分析

| 影响维度 | 分析 | 风险等级 |
|----------|------|----------|
| 上下游兼容性 | 现有价格查询接口无需修改，新增过滤逻辑在 Service 层封装 | 低 |
| 数据兼容性 | 新增 price_rule 表字段，需要数据迁移 | 中 |
| 配置变更 | 无需新增配置项 | 低 |
| 国际化 | 无用户侧展示变更，无需国际化 | 低 |
| 性能 | 过滤逻辑在内存中执行，需注意规则数量上限 | 中 |

## 验收标准

### 正常场景
1. Given 用户查询价格，When 存在匹配的渠道规则，Then 返回该渠道专属价格
2. Given 用户查询价格，When 存在匹配的用户等级规则，Then 返回该等级专属价格
3. Given 用户查询价格，When 同时存在多个规则，Then 按优先级返回最高优先级规则的价格
4. Given 用户查询价格，When 规则未在生效时间内，Then 不应用该规则

### 异常场景
1. Given 用户查询价格，When 无任何匹配规则，Then 返回默认价格
2. Given 用户查询价格，When 规则数量超过上限（100条），Then 抛出参数异常

### 边界条件
1. 优先级相同：按创建时间倒序，取最新的规则
2. 生效时间重叠：按优先级取最高的规则
3. 规则为空：返回默认价格，不报错

## 备注

- 价格规则最大数量限制为 100 条
- 规则过滤逻辑需要缓存优化，避免每次查询都查库
