## 链路 1：商品价格查询

### 触发条件
- 用户请求商品详情页
- 下游系统 RPC 查询商品价格

### 完整链路

```
[用户/下游系统]
    │
    ▼
┌──────────────────┐
│  Controller      │ 接收请求，参数校验
│  (app.controller)│ 校验 itemId 合法
└────────┬─────────┘
         │
         ▼
┌──────────────────┐
│  PriceService    │ 组装查询逻辑
│  (core.service)  │ 
└────────┬─────────┘
         │
         ▼
┌──────────────────┐
│  Cache Adapter   │ 查询 Tair 缓存
│  (integration)   │ key: price:{itemId}
└──────┬──┬────────┘
       │  │
   命中 │  │ 未命中
       │  │
       │  ▼
       │  ┌──────────────────┐
       │  │  PriceDAO        │ 查询 MySQL
       │  │  (dal.mapper)    │ SELECT * FROM price_rule
       │  └────────┬─────────┘
       │           │
       │           ▼
       │  ┌──────────────────┐
       │  │  Cache Adapter   │ 回写缓存
       │  │  (integration)   │ setex price:{itemId} 3600
       │  └────────┬─────────┘
       │           │
       ◄───────────┘
       │
       ▼
┌──────────────────┐
│  PriceService    │ 结果组装
│  (core.service)  │ 价格单位转换（分→元，仅展示层）
└────────┬─────────┘
         │
         ▼
┌──────────────────┐
│  Controller      │ 统一格式返回
│  (app.controller)│ Result<PriceVO>
└──────────────────┘
```

### 关键依赖
| 依赖 | 作用 | 降级策略 |
|------|------|----------|
| Tair 缓存 | 热点价格缓存 | 缓存不可用时降级为查库 |
| MySQL DB | 价格数据持久化 | 主库不可用切只读从库 |
| Diamond | 价格规则动态配置 | 本地缓存配置快照 |

### 性能指标
- P99 响应时间 < 50ms（缓存命中）/ < 200ms（缓存未命中）
- QPS 峰值：5000+
- 缓存命中率 > 90%

## 链路 2：价格规则变更

### 触发条件
- 运营后台新增/修改价格规则
- 定时任务触发的批量价格更新

### 完整链路

```
[运营人员 / 定时任务]
    │
    ▼
┌──────────────────┐
│  Controller      │ 接收变更请求
│  (web.controller)│ 参数校验 + 权限校验
└────────┬─────────┘
         │
         ▼
┌──────────────────┐
│  LiteFlow 流程   │ 流程编排
│  (core.flow)     │ 多步骤有序执行
└────────┬─────────┘
         │
    ┌────┼────┐
    ▼    ▼    ▼
┌─────┐ ┌─────┐ ┌──────────┐
│验证  │ │计算  │ │持久化    │
│组件1 │ │组件2 │ │组件3     │
└─────┘ └─────┘ └─────┬────┘
                       │
                       ▼
┌──────────────────┐
│  PriceDAO        │ 写入 price_rule 表
│  (dal.mapper)    │ 更新/插入规则记录
└────────┬─────────┘
         │
         ▼
┌──────────────────┐
│  Cache Adapter   │ 淘汰缓存
│  (integration)   │ del price:{itemId}
└────────┬─────────┘
         │
         ▼
┌──────────────────┐
│  PriceHistory    │ 记录变更历史
│  (core.service)  │ 写入 price_history
└────────┬─────────┘
         │
         ▼
┌──────────────────┐
│  MQ Producer     │ 发送价格变更通知
│  (integration)   │ 下游系统监听
└──────────────────┘
```

### 关键依赖
| 依赖 | 作用 | 降级策略 |
|------|------|----------|
| LiteFlow | 流程编排引擎 | 降级为纯 Service 串行调用 |
| Diamond | 读取规则配置 | 缓存本地配置 |
| MySQL DB | 规则持久化 | 事务失败回滚 |
| MetaQ | 变更通知 | MQ 不可用时降级为轮询 |

### 事务边界
- LiteFlow 主流程包含在数据库事务中
- 缓存淘汰在事务提交后执行
- MQ 消息在事务提交后发送（事务消息）

## 链路 3：配置动态更新

### 触发条件
- Diamond 配置中心配置变更

### 完整链路

```
[Diamond 配置变更]
    │
    ▼
┌──────────────────┐
│  ConfigListener  │ 监听配置变更事件
│  (bootstrap)     │ 
└────────┬─────────┘
         │
         ▼
┌──────────────────┐
│  配置更新处理器   │ 校验新配置
│  (core.service)  │ 格式验证 + 范围验证
└──────┬──────────┘
       │
       ▼
┌──────────────────┐
│  配置应用         │ 更新内存中的配置快照
│  (bootstrap)     │ 不需要重启生效
└────────┬─────────┘
         │
         ▼
┌──────────────────┐
│  配置缓存更新     │ 更新本地缓存
│  (common)        │ 
└──────────────────┘
```

### 配置项列表
| 配置 Key | 说明 | 变更影响 |
|----------|------|----------|
| price.discount.rate | 全局折扣率 | 实时影响价格计算 |
| price.cache.ttl | 缓存 TTL 秒数 | 影响缓存命中率 |
| price.rule.max_count | 单个商品最大规则数 | 影响规则校验 |
| price.switch.new_engine | 新价格引擎开关 | 功能灰度 |

## 高频变更区（Hot Paths）

以下链路是过去一年中变更频率最高的区域，Agent 在此类变更中需特别注意：

| 变更区域 | 频率 | 典型变更类型 | 注意事项 |
|----------|------|-------------|----------|
| 价格查询链路（缓存部分） | 高 | 缓存 Key 调整、TTL 优化 | 缓存一致性问题 |
| 价格计算规则 | 高 | 新增规则类型、折扣策略 | 兼容旧规则 |
| 外部对接接口 | 中 | 对接新渠道、接口字段扩展 | 兼容性保障 |
| 配置中心监听 | 中 | 新增配置项、监听逻辑修改 | 默认值设置 |
