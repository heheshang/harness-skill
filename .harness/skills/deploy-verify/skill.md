# Deploy Verify Skill

> 部署验证技能。用于部署验证阶段，确保部署正确、服务正常运行。

## 输入
- 构建产物（JAR/Docker Image）
- 部署环境信息
- 部署参数

## 输出
- 部署验证报告

## 执行步骤

### Step 1：部署前检查
- [ ] 编译产物正常
- [ ] 配置项已确认（与环境匹配）
- [ ] 数据库迁移脚本已就绪（如有）
- [ ] 回滚方案已准备

### Step 2：执行部署
> 部署通过 CI/CD Pipeline 自动触发，Agent 不应直接推送构建产物。
> Agent 的职责是：确认 Pipeline 已触发 + 监控部署状态。

1. 确认构建产物已在 CI pipeline 中生成
2. 确认部署 pipeline 已自动触发（或通过 CI MCP 触发）
3. 等待部署完成（监控 pipeline 状态）
4. 如 pipeline 失败，分析失败原因并回退到对应修复阶段

### Step 3：健康检查
```bash
# 健康检查端点
curl -f http://{host}:{port}/health

# 检查条件
- HTTP 200 OK
- 响应时间 < 1s
- 返回的 JSON 包含 status: "UP"
```

### Step 4：功能验证
- [ ] 核心接口正常返回 2XX
- [ ] 核心业务流程可正常执行
- [ ] 配置中心动态配置已生效
- [ ] 缓存预热完成

### Step 5：监控检查
- [ ] CPU 使用率正常（< 80%）
- [ ] 内存使用率正常（< 85%）
- [ ] GC 频率正常（无频繁 Full GC）
- [ ] 错误率正常（无异常增长）
- [ ] 接口响应时间正常（对比部署前基线）
